Shaku Gfx Demo: Performance Test

This demo is a performance test to test the limits of Shaku on your machine, with and without sprite batching optimization.
Note that it uses a single texture on purpose, to maximize the batching effectiveness; Using multiple textures decreases batching efficiency, which is why its important to use a texture atlas whenever possible.
A multi-texture performance test can be found here.

Press Z to add sprites, and X to remove them. Try to see performance with or without batching, the difference should be quite impressive.

The code below renders 100,000 sprites with batch rendering.

// load texture for test
let texture = await Shaku.assets.loadTexture('assets/skelly.png');

// create sprites group
let group = new Shaku.gfx.SpritesGroup();

// create 100,000 sprites
for (let i = 0; i < 100000; ++i) {
  let sprite = new Shaku.gfx.Sprite(texture);
  sprite.position.set(Math.random() * 800, Math.random() * 600);
  sprite.size.set(32, 48);
  sprite.color.set(Math.random(), Math.random(), Math.random(), 1);
  group.add(sprite);
}

// part below goes between startFrame() and endFrame()
// -------------------------------------------------------

// draw sprites with batching
Shaku.gfx.drawGroup(group, true);

// show current FPS count in element #show-fps
document.getElementById('show-fps').innerHTML = Shaku.getFpsCount();
